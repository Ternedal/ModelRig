# ModelRig – Brand Handoff v3

Dette er en mere komplet handoff-pakke for **ModelRig**.

Pakken er struktureret, så den kan bruges af:
- **design** (brandretning, logo, UI-retning)
- **produkt** (positionering, tone of voice, systemidentitet)
- **udvikling** (design tokens, implementeringsretning, UI-principper)
- **Figma / Claude / anden AI-assisteret produktion**

## Mappestruktur

### 01_logo-system
Indeholder oversigt over logosystemet:
- primært logo
- stacked logo
- standalone symbol
- monokrome varianter
- clear space / minimum size

### 02_app-icons
Indeholder app-ikon exploration:
- hovedikon
- light variant
- monokrom variant
- favicon / small mark
- launch / splash emblem

### 03_ui-concepts
Indeholder UI-koncept for ModelRig desktop-applikationen:
- premium dark UI
- navigation
- model-selector
- chat / RAG / dokumentkontekst
- model status og systemstatus

### 04_brand-guidelines
Indeholder overordnet brand guideline board:
- farvepalette
- typografi
- button styles
- input states
- ikonografi
- spacing / radius
- tone of voice

### 05_handoff-docs
Indeholder tekstlige dokumenter til handoff:
- brand summary
- implementeringsbrief
- Figma / design-system brief

## Anbefalet næste skridt
1. Tegn logo og symbol rent op som **vektor** i Figma eller Illustrator.
2. Opret **design tokens** direkte ud fra guideline-boardet.
3. Etabler et lille **komponentbibliotek** i Figma.
4. Eksportér app-ikonet i flere størrelser til desktop/app/web.
5. Brug implementeringsbriefen som handoff til udvikling.
